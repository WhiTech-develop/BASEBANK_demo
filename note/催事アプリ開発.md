# 面談議事録：ベースバンク（倉石様）システム構築案件

| 項目 | 内容 |
| ----- | ----- |
| **件名** | 催事買い取り事業の包括的な統合システム構築に関する要望ヒアリング |
| **日時** | 2026年3月5日 |
| **場所** | オンライン会議 |
| **出席者** | ベースバンク：倉石 凜太郎様 ホワイテック：吉村 佑紀 |
| **目的** | 倉石様からの新システム構築要望のヒアリング、および実現可能性・概算見積もり提示の依頼確認。 |

1\. 議論の概要ベースバンク様の事業背景と現状

* **事業内容:** 催事でのブランド品・時計などの買い取りと、仕入れた商品の販売チャネルでの販売を一気通貫で実施。  
* **売上規模:** 月間約3億円。  
* **現状の課題:** 現場のスケジュール管理（スプレッドシート）、在庫管理（Salesforce）、顧客との商談（LINE）など、データやプロセスが分散しており、現場管理から販売データまでを一元管理する統合システムをゼロから構築したい。

2\. ベースバンク様からの要望機能（統合システム）

ベースバンク様は、現場管理から着金までを一気通貫で完結できる「単一の自社アプリ」の構築を希望されています。

| 分野 | 要望機能の詳細 |
| ----- | ----- |
| **フロント（現場向け）** | 現場管理から顧客との商談、在庫管理、売上入金までのプロセスを単一アプリで完結。 |
|  | 顧客からの商品写真撮影と査定・価格付けのプロセス（現在LINEで実施）をシステム内部に取り込みたい。 |
|  | 現場の現金（資金）管理機能。 |
|  | 契約書の電子化、システムで作成した契約書を現場の持ち運びコピー機ですぐに印刷できる機能。 |
| **バックオフィス（本部向け）** | 査定担当者ごとの実績管理（査定商品、売上の伸び、偽造品発生率などの査定基準管理）。 |
|  | 販売先ごとの収益区分化と、前月比の売上増減を追跡できるデータ分析機能。 |
|  | 全社データを一元管理するデータベースの構築（既存のスプレッドシート、Salesforceデータを統合）。 |
| **システム要件** | 大容量データ（写真など）の無制限利用。 |
|  | 高速な処理スピード、シンプルで使いやすい操作性。 |
|  | **アプリケーション形式:** 見た目が「かっこいい」という理由から**ネイティブアプリ形式**（App Storeなどで公開されるアプリ）を希望。 |

3\. アプリケーション形式と利用規模に関する確認

* **アプリ形式:** 吉村氏（ホワイテック）からは、Webアプリ（ブラウザで利用）が主だが、ネイティブアプリ形式での開発も内容によっては可能であることを確認。  
* **ランニングコスト:** 現在の想定規模であれば、Webアプリとネイティブアプリでのランニングコストに大きな差はない見込み。  
* **利用規模:** 約200人規模での利用を想定（常時ログインもそれに近い形）。  
* **既存データ管理:** 現在、スケジュール・実績管理をスプレッドシート、在庫管理をSalesforceで実施。これを新システムで一元管理したい。  
* **利用開始時期:** 早ければ早いほど良い。
