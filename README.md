## 1. コンセプト概要

### このシステムが解決する課題

貴金属・ブランド品の催事買取ビジネスは、現場と本部が**リアルタイムに連携できるかどうか**で、1日あたりの成約件数と粗利が大きく左右されます。

従来の業務フローでは、以下の構造的な問題がありました。

| 課題 | 現状の影響 |
|------|-----------|
| LINEやスプレッドシートでの情報共有 | 査定依頼の見落とし・情報の散逸 |
| 画像送信の遅延（数十秒〜数分） | 顧客を長時間待たせ、成約率が低下 |
| 本部への確認が属人的 | 特定の担当者不在時に査定業務が止まる |
| 成約データの事後入力 | 集計のズレ・在庫管理の不整合 |

### BASEBANK System が提供する価値

> **「現場と本部を、単一のリアルタイムプラットフォームで繋ぐ」**

本システムは、現場バイヤーが商品登録・写真撮影を行うと、**即座に本部の査定コンソールへ通知が届き**、査定価格が返答された瞬間に現場に表示されます。商談の速度を最大化し、顧客体験と成約率を同時に向上させます。

## 2. 主要機能とベネフィット

### 現場バイヤー向け機能

| 機能 | 説明 | ユーザーが得る利益 |
|------|------|--------------------|
| **業務開始セットアップ** | 担当会場・準備金を入力してシフト開始 | 「誰がどの会場にいるか」が本部に即時共有され、管理コストがゼロに |
| **リアルタイム買取ダッシュボード** | 本日の買取総額・手元残高をKPI表示 | 現場判断（あと何件買取できるか）が数値に基づいて行え、機会損失を防ぐ |
| **商品カテゴリ別登録** | 金・宝飾・ブランド・時計・その他をタブで切替、重量・状態を入力 | 入力項目が最適化されており、1件の商品登録を最短操作で完了できる |
| **写真アップロード** | 商品画像を撮影・ドラッグ&ドロップでS3に即送信（Transfer Acceleration対応） | 高精細画像が本部に数秒以内に届き、査定精度と速度が向上 |
| **査定依頼送信** | ワンタップで本部へ査定リクエスト送信 | 電話・LINEでの連絡が不要になり、現場スタッフの集中力が商談に向く |
| **本部回答のリアルタイム表示** | 査定価格・コメントが画面にプッシュ表示 | 顧客を待たせる時間が激減し、商談がスムーズに進む |
| **成約・不成約の即時記録** | ボタン一つで結果を確定・集計に反映 | 事後のデータ入力作業が消滅し、リアルタイムで本部が全体状況を把握できる |
| **電子署名（拡張予定）** | 成約時に顧客署名をその場でデジタル収集 | 書類管理・郵送コストがゼロになり、コンプライアンスリスクも低減 |

### 本部査定員向け機能

| 機能 | 説明 | ユーザーが得る利益 |
|------|------|--------------------|
| **査定キュー管理** | 全会場からの依頼が一覧で届き、未対応件数をバッジ表示 | 依頼の見落としがなくなり、ピーク時でも対応漏れゼロを実現 |
| **商品詳細・写真の即時確認** | 現場からの写真・商品情報を高解像度で表示 | 実物を見なくても的確な査定が可能になり、遠隔での業務完結率が上がる |
| **買取価格・販売想定額の入力** | 下代（買取価格）と上代（販売想定）を入力して返答 | 粗利計算が自動化され、利益を確保しながら速やかに回答できる |
| **ダッシュボードKPI** | 未対応件数・本日の粗利目標をリアルタイム表示 | 経営目標と現場進捗が常に可視化され、リソース調整の判断が瞬時にできる |

## 3. ユーザージャーニー（モック体験フロー）

### 現場バイヤー側フロー

```
[1] ログイン
      ↓ 担当会場・準備金を入力
[2] ダッシュボード確認（本日の買取総額・残高）
      ↓ 新規顧客ボタンをタップ
[3] 顧客情報入力（氏名・来店経路）
      ↓ 商品カテゴリを選択
[4] 商品登録（写真撮影・重量・状態コメント入力）
      ↓ 「査定依頼を送る」ボタン
[5] 査定待ち画面（スピナー表示）
      ↓ 本部から回答が届くとプッシュ表示
[6] 査定回答表示（買取価格・コメント）
      ↓ 顧客が承諾
[7] 成約 → 電子署名 → 買取完了
      ↓ ダッシュボードの残高・総額が即時更新
```

### 本部査定員側フロー

```
[1] ダッシュボード（未対応件数をKPI確認）
      ↓ 「査定コンソール」タブへ
[2] 依頼キューから案件を選択
      ↓ 商品写真・詳細情報を確認
[3] 買取価格・販売想定・コメントを入力
      ↓ 「査定回答を送信」ボタン
[4] 次の案件へ自動移行
      ↓ 現場へリアルタイムに通知が届く
```

│  │   Cognito     │   │    （GraphQL API + WS）       │ │
│  │  （認証・認可）│   │  Queries / Mutations /        │ │
│  └──────────────┘   │  Subscriptions               │ │

│  ┌──────────────┐                  │                  │

### 各技術選定の根拠

#### AWS Amplify（サーバーレスフレームワーク）
- インフラ管理なしで認証・API・ストレージ・リアルタイム通信を統合
- スタッフ数や催事件数の増減に応じて**自動スケール**（固定インフラコスト不要）
- 将来のモバイルアプリ・分析機能追加もフレームワーク内で拡張可能

#### AWS AppSync（GraphQL API）
- **WebSocketベースのリアルタイム通信**をマネージドで提供
- 現場と本部が同一データソースを双方向に監視できる設計（Subscriptions）
- クライアントが「必要なデータだけ」を取得するGraphQL構造により、通信量・遅延を最小化

#### Amazon DynamoDB（NoSQLデータベース）
- **個位ミリ秒のレスポンスタイム**を保証（催事のピーク時間帯でもパフォーマンス劣化なし）
- テーブル設計済み：Deal / Item / Message / Staff / Venue

#### Amazon S3 + Transfer Acceleration
- 商品画像を**世界最速経路（AWS エッジネットワーク経由）でアップロード**
- 大阪リージョンのバケットへ東京・大阪どこからでも高速送信
- Transfer Acceleration により、通常S3比で最大**60〜500%の転送速度向上**

#### 大阪リージョン（ap-northeast-3）採用
- 主要催事エリア（関西・東海）との物理的距離を最短化
- 東京リージョンと比較して**往復レイテンシを数ms単位で削減**
- データの国内完結（GDPR・個人情報保護法への対応）

#### Vanilla JS（フレームワークレス）
- ビルド環境不要 → **デプロイが静的ファイル配信のみ**（CloudFront等に即対応）
- AWS Amplify SDKとの直接統合が容易
- 本番移行時に任意のフレームワーク（React / Vue等）への段階的移行が可能

## 5. AWS 環境構築状況

### 現在のインフラ構成

AWS 環境は既に構築済みです。モック段階でも本番同等のクラウドサービスに接続されています。

| サービス | 構築状況 | 詳細 |
|---------|---------|------|
| **AWS Amplify** | 構築済み | プロジェクト初期化・設定完了 |
| **AWS AppSync** | 構築済み・稼働中 | GraphQLエンドポイント発行済み（大阪リージョン） |
| **Amazon Cognito** | 構築済み・稼働中 | ユーザープール・IDプール作成済み、メール認証有効 |
| **Amazon DynamoDB** | 構築済み | スキーマ定義済み（Amplify管理下） |
| **Amazon S3** | 構築済み・稼働中 | バケット作成済み・Transfer Acceleration有効 |
| **AWS Lambda** | 定義済み | AppSync Resolverとして連携設定済み |

| 項目 | 現状 | 本番想定 |
